# デモ環境のセットアップ手順

このアプリを「制作サンプル」としてホームページに掲載するための、**本番とは完全に分離した安全なデモ環境**の作り方です。実在の会員情報は一切含まれません。

## 仕組み

- デモ用に **別のSupabaseプロジェクト**（ダミーデータ）と **別のVercelデプロイ** を用意します
- デモデプロイには環境変数 `DEMO_MODE=true` を設定します
- `DEMO_MODE` が有効だと、API側で**データの追加・変更・削除がすべてブロック**され、閲覧のみ可能になります（誰かがデモを壊せません）
- 画面上部に「これはデモ版です」バナー、ログイン画面にデモ用ID/パスワードが表示されます

## 手順

### 1. デモ用Supabaseプロジェクトを作成
1. Supabaseで新しいプロジェクトを作成（本番とは別プロジェクト）
2. SQL Editor で `supabase/migrations/001_initial.sql` を実行（スキーマ作成）
3. 続けて `supabase/seed_demo.sql` を実行（ダミーデータ投入）

### 2. デモ用Vercelデプロイを作成
同じGitHubリポジトリから、もう1つVercelプロジェクトを作成し、以下の環境変数を設定します。

| 変数名 | 値 |
|---|---|
| `SUPABASE_URL` | **デモ用**SupabaseのURL |
| `SUPABASE_SERVICE_ROLE_KEY` | **デモ用**Supabaseのservice_roleキー |
| `DEMO_MODE` | `true` |

> VAPIDキー（プッシュ通知）やLINE関連はデモでは不要です（未設定でも動作します）。

### 3. デプロイして確認
- デモ用URLにアクセス → ログイン画面にデモ用の認証情報が表示されます
- 管理者：`山田 太郎` / `demo1234`
- 一般：`鈴木 花子` / `demo1234`

## 注意

- **本番のSupabaseに `seed_demo.sql` を実行しないでください**（`TRUNCATE` で全データが消えます）
- 本番デプロイには `DEMO_MODE` を設定しないでください（設定すると本番でも書き込みできなくなります）
- デモが荒らされた場合は `seed_demo.sql` を再実行すればリセットできます
