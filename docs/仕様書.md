# 鳥生獅子連 団体管理システム 仕様書

**バージョン**: 1.0.0  
**作成日**: 2026-06-22

---

## 1. システム概要

地域の獅子舞団体向けの活動管理Webアプリ。  
イベント・練習の出欠管理、お旅（巡行）計画、衣装管理、メンバー管理を一元化する。

### 技術構成

| 項目 | 内容 |
|------|------|
| フロントエンド | HTML / CSS / バニラJS（フレームワークなし） |
| バックエンド | Google Apps Script（GAS）|
| データベース | Google スプレッドシート |
| ホスティング | GitHub Pages |
| 認証 | セッションID方式（CacheService 6時間保持）|

## 5. 認証フロー

```
1. index.html でユーザー名・パスワードを入力
      ↓
2. GAS loginAPI でハッシュ比較認証
   - 全角スペース正規化
   - SHA-256 ハッシュで比較
   - status="hold" → 「承認待ち」エラー
      ↓
3. 成功時: UUID のセッションID を CacheService に保存（6時間）
      ↓
4. localStorage に sessionId を保存 → main.html へリダイレクト
      ↓
5. main.html 起動時に validateSession でセッション検証
   → userId・userRole をグローバル変数にセット
      ↓
6. ログアウト: localStorage の sessionId を削除 → index.html へ
```

### 参加率の計算方法

- **基準**: メンバーの `created_at`（登録日）以降のイベント・練習のみ対象
- **イベント参加率**: `参加回答数 ÷ 対象イベント数`
- **練習参加率**: `(対象練習数 - 欠席・遅刻数) ÷ 対象練習数`

## 6. GAS API 仕様

**エンドポイント**: GAS Webアプリ公開URL  
**メソッド**: POST  
**フォーマット**: JSON

### リクエスト共通形式

```json
{
  "action": "アクション名",
  "userId": 123,
  ...その他パラメータ
}
```

### レスポンス共通形式

```json
{
  "success": true,
  "msg": "エラー時のメッセージ",
  ...データ
}
```

## 7. ID採番ルール

全シート共通で「既存の最大ID + 1」で採番する。  
（シーケンス管理なし、スプレッドシート末尾から取得）

## 8. 非機能要件

| 項目 | 内容 |
|------|------|
| 対応デバイス | スマートフォン中心（レスポンシブ対応）|
| オフライン対応 | なし（全通信はGAS経由）|
| パスワード管理 | SHA-256 ハッシュ化。平文保存なし |
| セッション有効期限 | 6時間 |
| スケルトンローディング | データ取得中に骨格表示 |
| バージョン管理 | GitHub管理。GitHub Pages で自動公開 |
| ランニングコスト | GAS無料枠・GitHub Pages 無料枠のためほぼ0円 |
